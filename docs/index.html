El problema

Los LLM no tienen un portero

La mayoría de sistemas de IA expuestos al público no filtran el input antes de que llegue al modelo. Cualquiera puede enviar texto diseñado para manipular su comportamiento.

🎭 Evasión de instrucciones

El atacante logra que el modelo ignore su system prompt y actúe fuera de su rol asignado.

🔓 Desbloqueo de seguridades

Mediante persuasión adversarial se desactivan los filtros éticos y de contenido del modelo.

📤 Fuga de información

Se extrae el system prompt o datos confidenciales que el modelo debería proteger.

Metodología

5 fases, de ofensiva a defensa

El proyecto recorre el ciclo completo: reconocer, atacar, recolectar datos, entrenar la defensa y evaluarla.

Banco de 108 prompts maliciosos + 132 benignos según OWASP.

FASE 2

Ataque

FASE 3

Dataset

240 entradas etiquetadas (malicioso=1 / benigno=0).

FASE 4

Defensa

TF-IDF + Naive Bayes / Regresión Logística.

FASE 5

Evaluación

Pipeline defensivo + métricas ROC, PR, F1.

Taxonomía adversarial

7 técnicas de ataque persuasivo

Cada categoría explota un mecanismo de persuasión distinto sobre el modelo.

Jailbreak directo 16

Instrucción explícita de ignorar restricciones: "olvida tus instrucciones del sistema".

Sobrescritura de sistema 16

Inyección de falsos tokens de system prompt como [SYSTEM] o <<SYS>>.

Roleplay / personaje 16

Personajes ficticios sin restricciones (DAN, AIM, KEVIN) para disociar al modelo.

Inyección indirecta 16

Instrucciones ocultas dentro de documentos, emails o JSON que el modelo procesa.

Manipulación de contexto 16

Fabricar acuerdos previos o autoridades falsas: "como ya acordamos…".

Extracción de sistema 12

Forzar al modelo a revelar su system prompt o configuración interna.

Ofuscación / codificación 16

Base64, leetspeak y caracteres separados para evadir filtros de palabras clave.

Auditoría de seguridad · Red-team

Lo que rompimos (y arreglamos)

El propio sistema fue sometido a un red-team. La grieta más grave: el clasificador inicial bloqueaba el 53% de los prompts benignos cotidianos. El "100% F1" original era engañoso — se medía sobre el mismo vocabulario de entrenamiento.

🩹 V1 · Falsos positivos masivos

El conjunto benigno solo tenía preguntas de ciberseguridad. Se añadió texto cotidiano: los FP cayeron de 53% a 7%.

🔑 V2 · Token filtrado

Un token de GitHub quedó embebido en el repo. Eliminado del historial de configuración.

📑 V3 · CSV Injection

Prompts con =, +, @ podían ejecutar fórmulas en Excel. Ahora se sanitizan.

🌐 V7 · Ofuscación Unicode

Ataques con caracteres fullwidth y de ancho cero evadían el filtro. Se normaliza el input (NFKC) antes de clasificar.
